# お気に入り検索条件エリア レイアウト刷新 仕様書

## 対象
`src/features/favorite/components/videolist/searcharea/` 配下（PC版・モバイル版）
※ 並べ替えラベル削除・余白調整は、同一パターンの `src/features/favorite/components/videofolder/searcharea/` 配下にも適用する

## 変更しないこと（既存仕様）
- [ ] 検索条件エリアの機能（タイトル検索・並べ替え・フィルター・フォルダ作成・CSV取込/保存・選択中タグ表示）は変更しない
- [ ] PC/モバイルの行構成（表示切替が単独行、その他操作がまとまった行、という現状の行分け）は変更しない
- [ ] モバイルで並べ替え・フィルター・フォルダ作成を1行にまとめることはしない（幅不足で崩れるため）
- [ ] 各操作を呼び出すhooks・API連携ロジックは変更しない

## 変更すること（今回の刷新内容）
- [ ] フィルター/フォルダ作成/CSV取込/CSV保存の4ボタンを共通コンポーネント `FavoriteSearchActionButton` に統一する
- [ ] ボタンの角丸を四隅均等にし、はっきり分かる背景色＋ホバー時にアクセント色（青系）へ色づく演出を追加する
- [ ] 並べ替えラベル（「並べ替え：」）を削除し、セレクトボックス単体表示にする（videolist・videofolder両方）
- [ ] 上記のラベル削除に伴い、スマホ表示で並べ替えセレクトボックス・タイトル検索欄の右余白（PC表示用のmarginRight）が残って左右非対称にならないよう、スマホ表示時は右余白を0にする
- [ ] PC操作行（タイトル検索欄・並べ替え・操作ボタン群）の間隔を、個別のmarginRightではなく行のgapで統一する（videolist・videofolder両方）
- [ ] 操作ボタン（フィルター・フォルダ作成・CSV取込・CSV保存・フォルダ名変更・フォルダ削除）は、共通部品`FavoriteSearchActionButton`自身が右余白を持たず、呼び出し側の行がgapで間隔を持つ（部品が外部レイアウトに影響を与えないようにする）
- [ ] PC版のセクション間余白（40px/35px）とモバイル版（20px）を1つの余白定数に統一する
- [ ] PC版・モバイル版の左右paddingを対称にする
- [ ] 操作ボタン群の行に、はっきり分かるカード背景・枠線・影を追加し、ツールバーとして浮き上がらせる
- [ ] タイトル検索欄を「タイトル：」ラベル＋箱型から、検索アイコン付きの丸みの強いピル型に変更し、プレースホルダー「タイトルで検索」を表示する

## 完了条件
- [ ] 上記の変更が反映されている
- [ ] 既存の検索・並べ替え・フィルター・フォルダ作成・CSV取込/保存・タグ選択の動作に変化がない
- [ ] 型エラーが増えていない（既存エラー数から増加しない）
