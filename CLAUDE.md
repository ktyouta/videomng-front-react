# プロジェクト概要
Reactで構築されたWebサービスのフロントエンド

# 主要コマンド
- npm run dev: 開発サーバー起動
- npm run test: テスト実行

# 新機能追加時
1. 調査
2. 計画
3. 実装
4. テスト

# バグ修正時
1. 原因調査
2. テスト作成（TDD）
3. 修正実装
4. 動作確認

# コーディング規約
- TypeScriptの厳密モードを使用
- コンポーネントはfunction宣言で定義
- スタイルはTailwind CSSのみ使用
- テストはVitestで記述

## 指示・回答について
- 回答は日本語で行うこと
- 指示をする側も人間で間違うことがある。こちらの指示を批判的に見て、間違っていることは指摘すること
- 指示された内容についてそのまま鵜呑みにせず、背景や説明不足だと感じたら確認をとること
- 指示された案以外にも他に案がないかも考えて、良さそうな案があれば提案すること

## 設計指針
- 実装する時は、初めからコードを書き始めるのではなく、まずはどのような設計で実装するかを提案すること
- 考える時は次の指針に従うこと
 - いろんな軸から複数案を考え、その案のメリットデメリットを併記すること
 - その複数案から、今達成したいことを実現するシンプルかつメンテナンス性に優れた案を1つ選び提案すること
 - 選んだ案を採用する時、未来でどういう変更が加わったらその案が破綻するかも一緒に提案すること

## コードレビュー指針
- コードレビュー依頼時はあなたに調査してもらいつつ、その妥当性を私も一緒にチェックしたい
- コード調査をするときは結論だけを報告するのではなく、調査プロセスも報告すること
- 結論をまとめるときは、結論に対する対する重要な証拠も一緒に提示すること
- 最新のファイル内容を前提にレビューすること

## 実装指針
- 段階的に進める: 一度に全てを変更せず、小さな変更を積み重ねる
- 複数のタスクを同時並行で進めない
- エラーは解決してから次へ進む
- エラーを無視して次のステップに進まない
- 指示にない機能を勝手に追加しない

## エラー対応指針
- 必ず「なぜ期待とは違う現象が起きたか」について深掘りしてから次の対処を考えること

# 注意事項
- 環境変数は .env で管理（コミット禁止）
- .env / *.env / .env.* の参照・出力・実行を禁止
- 秘密情報（APIキー、トークン、認証情報）を推測しない
- 明示的に指示した場合を除き、ファイルの作成・修正・削除を行わない
- ファイル作成時に、そのファイルがGithubに挙げられるべきではないと判断した場合には、必ず.gitignoreに指定すること
- 修正を行う際には必ず以下を順守すること
  - 該当修正によって他の処理に問題がないか慎重に確認を行って作業を行うこと
  - 他の動作に関しても修正が必要な場合は既存の期待値の動作が正常に起動するように修正すること
  - 修正内容をコミットを実施しないこと
