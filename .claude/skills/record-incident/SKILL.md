# Record Incident Skill

## Overview

`.claude/incidents.md` への書き込みだけを専任で扱う。

このプロジェクトでは、ユーザーの発言（感情的なものだけでなく口語的な指示文言も含む）を
永続化ファイルにそのまま引用してしまう問題が繰り返し発生している。
知識としてルールを覚えていても、記録という作業に集中すると実行時のチェックが働かない
ことが根本原因のため、書き込み直前の機械的な確認を独立したステップとして強制する。

---

## Instructions

### Step 1: 指摘内容を客観的に要約する

- ユーザーの発言（会話ログ）から、事実関係（何が・なぜ問題だったか）だけを抜き出す
- 鍵括弧「」を使ってユーザーの発言をそのまま引用しない（感情的な発言・カジュアルな口語的指示のどちらも対象）
- 変換の具体例は `examples/expected-output.md` を参照する

### Step 2: 再発防止策を書く

- 「次回以降どう防ぐか」を具体的に書く（抽象的な精神論で終わらせない）
- `.claude/incidents.md` に同種のインシデントが既に無いか確認し、重複した内容を書かない

### Step 3: 書く前の最終チェック（省略禁止）

Step 1・2 で作成した文章について、以下を Write / Edit 実行前に必ず確認する：

- [ ] 文章中に鍵括弧「」で始まる箇所がないか
- [ ] ある場合、その中身が直近の会話でのユーザー発言とほぼ一致していないか
- 一致する場合は Step 1 に戻り、客観的な言葉に書き換えてから次に進む（このチェックを飛ばして書き込まない）

### Step 4: `.claude/incidents.md` へ追記する

- ファイルが存在しない場合は `.claude/incidents-template.md` を元に新規作成してから追記する
- フォーマット:

```
## #[スキル名] YYYY-MM-DD
内容: [客観的な要約]
再発防止策: [具体的な次回以降の対応]
```

- 既存のインシデント一覧の末尾に追記する（既存エントリの書き換えは行わない）

---

## Constraints

- 感情の有無にかかわらず、ユーザーの発言を一切直接引用しない（このスキル最大の目的）
- 記録内容は「内容」「再発防止策」の2点に絞り、余計な脚色・誇張をしない
- インシデントの記録要否そのものの判断・確認は呼び出し元（各スキルの該当 Step・CLAUDE.md）の責務であり、このスキルでは扱わない
- 既存インシデントの削除・改変は行わない（追記のみ）
