# Commit Message Skill

## Overview

実装完了後に、過去のコミット履歴のスタイルを参照して一貫性のあるコミットメッセージを提案する。

**重要：このスキルはメッセージを提案するだけであり、`git commit` / `git add` などのコミット操作は絶対に行わない。コミットはユーザーが自分で実施する。**

---

## Procedure

1. `git log --oneline` で全コミットメッセージを確認し、スタイル・語調・粒度を把握する
2. `git diff HEAD` で変更内容を確認する
3. 複数の機能・概念にまたがる変更がある場合は、最も修正量が多い／中心的な機能1つに着目し、そこを主語にメッセージを構成する（すべての変更を列挙しない）
4. 過去のスタイルに沿ったコミットメッセージを「詳細版」と「簡易版」の2グループで5件ずつ作成する
   - 詳細版：動詞 + 何を + どんな状況・条件（例: お気に入り一覧のページネーションが1ページ目に戻らない不具合を修正）
   - 簡易版：動詞 + 何をしたか のみ（例: 動画詳細画面にコピーボタンを追加）
   - 各候補には、なぜそのメッセージが適切かの根拠を1行で添える
5. **英語の場合は、各候補に必ず日本語訳を括弧書きで付ける（訳の記載がない候補は不完全とみなす）**
6. Output Format に従って出力する（形式が合っているか出力前に確認する）
7. コミット操作は一切行わない

---

## Output Format

英語の場合：

```
## コミットメッセージ候補

### 詳細版
1. [英語メッセージ]（[日本語訳]）
   → [根拠]
2. ...（計5件）

### 簡易版
1. [英語メッセージ]（[日本語訳]）
   → [根拠]
2. ...（計5件）
```

日本語の場合：

```
## コミットメッセージ候補

### 詳細版
1. [候補1]
   → [根拠]
2. ...（計5件）

### 簡易版
1. [候補1]
   → [根拠]
2. ...（計5件）
```
