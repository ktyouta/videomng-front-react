# Spec Review Skill

## Overview

設計書と実装コードを照合し、仕様との差分を検出する。
「コードの品質」ではなく「仕様との適合」を確認することが目的。

---

## Procedure

1. `docs/[機能名]/plan.md` を読み込む
2. 実装済みのコードを読み込む（変更されたファイルを対象とする）
3. 設計書から要件項目を抽出する
4. 要件項目ごとに実装の有無・内容を照合する
5. 以下の形式で報告する

---

## Output Format

違反がある場合：

```
## 仕様突き合わせ結果

### 実装済み（仕様通り）
- [要件項目]: [該当ファイル:行番号]

### 未実装
- [要件項目]: 実装が確認できない

### 仕様と異なる実装
- [要件項目]
  - 仕様: ...
  - 実装: [ファイル:行番号] ...
  - 差分の概要: ...
```

違反がない場合：

```
## 仕様突き合わせ結果

チェック完了。仕様通りに実装されています。
```

---

## Constraints

- コードの品質・パターンは対象外（それは frontend-review が担う）
- spec.md がまだ無い場合、このスキルでは新規作成しない（呼び出し元スキルの該当 Step で作成する）
- 「仕様と異なる」の判定は明確な差分があるときのみ行う（曖昧な場合はユーザーに確認する）
