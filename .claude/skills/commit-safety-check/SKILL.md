# Pre-Commit Check Skill

## Overview

実装完了後またはコミットメッセージ依頼時に、変更ファイルに機密情報が含まれていないかチェックする。

**重要：このスキルはチェックと報告のみを行う。`git commit` / `git add` 等の操作は一切行わない。**

---

## Check Instructions

### 機密ファイルの混入確認
以下のファイルが変更対象に含まれていないか確認する：
- `.env` / `.env.*`
- `*.pem` / `*.key` / `*.p12` / `*.pfx`
- `*secret*` / `*credential*` という名前のファイル

### ハードコードされた機密情報
変更ファイルの差分（追加行）に以下のパターンがないか確認する：

| カテゴリ | 検出パターン例 |
|---|---|
| API キー | `sk-`、`ghp_`、`ghs_`、`AIza`、`AKIA` |
| パスワード | `password\s*=\s*["'][^"']+["']`、`passwd\s*=` |
| シークレット | `secret\s*=\s*["'][^"']+["']`、`SECRET\s*=` |
| 秘密鍵 | `-----BEGIN (RSA |EC |OPENSSH )?PRIVATE KEY-----` |
| JWT | `eyJ[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+` |
| トークン | `token\s*=\s*["'][A-Za-z0-9+/]{20,}["']` |

### ローカル環境依存パス（**絶対禁止**）
以下のパターンがコード・ドキュメントのいずれに含まれていても**コミット不可**とする：

| カテゴリ | 検出パターン例 |
|---|---|
| Windows 絶対パス | `C:\`、`D:\`、`C:/Users/` 等のドライブレター始まりのパス |
| Unix ホームディレクトリ | `/home/[username]/`、`/Users/[username]/` |

**なぜ禁止か**: ローカルパスはコードやドキュメントを他の環境で動かせなくする。チームで共有するファイルにはプロジェクトルートからの相対パスのみ使用すること。

---

## Procedure

1. `git diff HEAD --name-only` で変更ファイル一覧を確認する
2. 機密ファイル名が含まれていないか照合する
3. `git diff HEAD` で追加行の内容を確認し、以下を照合する
   - 機密情報パターン
   - ローカル環境依存パス（`C:\`、`D:\`、`/home/`、`/Users/` 等）
4. 新規追加ファイル（`git status` で `??` のもの）がある場合は、その内容も同様にチェックする
5. 以下の形式で報告する

---

## Output Format

問題がある場合：

```
## Pre-Commit Check 結果

### 要確認
- **ファイル**: [ファイルパス:行番号]
- **検出内容**: 具体的な検出パターン
- **対応方針**: 環境変数化・ファイル除外等
```

問題がない場合：

```
## Pre-Commit Check 結果

チェック完了。問題なし。
```
