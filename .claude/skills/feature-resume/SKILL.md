# Feature Resume Skill

## Overview

git の状態と設計書を読み取り、前回の作業がどこまで完了しているかを判断して報告する。
feature-impl の途中で止まった作業を、正しいステップから再開することが目的。

---

## Procedure

### Step 1: git の状態を確認する

以下を実行する：

- `git log --oneline -10` — 直近のコミット履歴を確認
- `git status` — 未コミットの変更を確認
- `git diff --stat` — 変更ファイルの概要を確認

---

### Step 2: 設計書を確認する

`docs/` 配下の `plan.md` を確認し、何の機能を実装しようとしていたかを把握する。

---

### Step 3: 進捗を判断する

Step 1・2 の情報をもとに、feature-impl のどのステップまで完了しているかを判断する。

| 状態 | 判断基準 |
|---|---|
| Step 3 未完 | plan.md はあるが、実装のコミットがない |
| Step 4〜5 完了 | 実装・レビューのコミットがある、仕様突き合わせ（Step 6）未実施 |
| Step 6〜7 完了 | 仕様突き合わせ・plan.md 更新が済んでいる、NG 対応（Step 8）が残っている可能性がある |
| 完了 | 全ステップのコミットがあり、未コミット変更がない |

---

### Step 4: 結果を報告する

```
## 作業再開レポート

### 実装中の機能
- [機能名]: docs/[機能名]/plan.md より

### 完了済みステップ
- [完了しているステップを列挙]

### 再開ポイント
- feature-impl の Step [N] から再開してください
- 理由: [git log / git status から読み取った根拠]

### 未コミットの変更
- [あれば列挙、なければ「なし」]
```

---

## Constraints

- git コマンドの実行結果を根拠として示す（推測で判断しない）
- 判断が難しい場合はユーザーに確認する
- 実装は行わない（再開ポイントの報告のみ）
