# Session Retrospective Skill

## Overview

セッション末尾に会話を振り返り、既存スキルへの改善提案を出す。

スキルの自律的な進化を促すことが目的。
提案のみ行い、ファイルの作成・修正は一切行わない。

---

## Instructions

### Step 1: SKILL.md Phase 突合

今回の会話で呼び出されたスキルを特定し、その SKILL.md を Read で読み込む。
読み込んだ SKILL.md から Phase（手順・ステップ）の一覧を抽出し、
会話中のツール呼び出し履歴と1:1で突合する。

以下の3種類を検出する：

**スキップされた Phase**
- SKILL.md に記載があるのに、対応するツール呼び出しが会話中に見当たらない
- → 「条件付き実行に変更すべきか」「常に必要なのにスキップされたか」を判定する

**AI が自己追加した Phase**
- SKILL.md にない手順を AI が独自に実行した
- → 再現性があれば SKILL.md への追記候補とする

**順序の入れ替え**
- SKILL.md の順序と異なる順で Phase を実行した
- → 実際の順序の方が効率的だったかを判定する

突合結果は Step 2 の入力として使う。

---

### Step 2: 今回の会話を振り返る

以下の観点で会話全体を確認する。

**ユーザーの修正指摘**
- 同じ内容の指摘が複数回あったか
- 「毎回これを言っている」「また同じミス」のような言葉があったか

**Claude の迷い・確認の多発**
- 設計判断でユーザーに繰り返し確認を求めたか
- 説明に詰まった箇所があったか

**既存スキルとのずれ**
- 既存スキルの範囲外の判断を行ったか
- トリガーワードが機能しなかった・スキルを呼ぶべき場面で呼ばなかったか

**実装の漏れ・やり直し**
- レビュースキルで検出できなかった問題が発生したか
- 水平展開の漏れが後から発覚したか

---

### Step 3: パターンを判定する

以下の条件を満たすものだけを対象にする。

- 1回限りの判断ミスは対象外（再現性があるものだけ）
- 「おそらく」レベルの推測は対象外（明確に観察できたものだけ）
- 既にスキルに記載済みの内容は対象外

---

### Step 4: 提案を絞る

最大3件に絞る。優先度の高いものから選ぶ。

優先度の基準：
1. ユーザーが明示的に指摘したもの（最優先）
2. 実装の漏れ・やり直しに繋がったもの
3. Phase 突合で検出したスキップ・追加・順序変更
4. Claude が迷った・確認が多発したもの

---

### Step 5: 結果を出力する

examples/expected-output.md の形式で出力する。

パターンが見当たらない場合は何も出力しない。

---

### Step 6: パターンを memory に保存する

Step 5 で提案を出力した場合のみ実行する。

以下のルールで memory に記録する：

- memory ファイル（`session-retrospective-patterns.md`）が存在しない場合は新規作成する
- 既存ファイルがある場合は該当パターンのカウントを更新する
- 保存形式:

```md
| パターン | 対象 Skill | 発生回数 | 最終検出日 |
|---|---|---|---|
| [パターン概要] | [skill名] | [n] | [YYYY-MM-DD] |
```

発生回数が2以上になったパターンがある場合、detect-recurring-feedback を呼び出す。

---

### Step 7: 未記録インシデントの確認

セッション中にユーザーから指摘・修正があったにもかかわらず、`.claude/incidents.md` への記録確認を行わなかったケースがないか振り返る。

該当するケースがあれば、以下の確認を行う：
「今回の指摘を #[スキル名] のインシデントとして記録しますか？（記録内容: [指摘の要約を1行で]）」

---

## Constraints

- スキルのファイル作成・修正は一切行わない（提案のみ）
- パターンが見当たらない場合は出力しない（ノイズを出さない）
- 最大3提案まで（過多にならないよう絞る）
- 「おそらく」「たぶん」レベルの推測を提案に含めない
